Guía de Pacman y Yay en Arch Linux

Pequeña guía para recordar los comandos que más utilizo en Arch.

Gestión de Paquetes con Pacman

Instalar Paquetes

pacman -S package_name1 package_name2 ...

Este comando instala uno o más paquetes. Puedes listar múltiples paquetes separados por espacios.

Eliminar Paquetes

pacman -R package_name

Elimina un paquete instalado.

pacman -Rn package_name

Elimina un paquete y sus archivos de configuración.

pacman -Rs package_name

Elimina un paquete y sus dependencias que no son requeridas por otros paquetes.

pacman -Rns package_name

Combinación para eliminar completamente un paquete, sus configuraciones y dependencias no necesarias.

Limpiar Dependencias no Utilizadas

pacman -Qdtq | pacman -Rs -

Este comando elimina las dependencias que ya no son requeridas por ningún paquete instalado (equivalente a 'autoremove' en Ubuntu).

Actualizar Paquetes

pacman -Syu

Actualiza todos los paquetes instalados. Aquí, 'S' significa sincronizar, 'y' refresca la caché local, y 'u' actualiza el sistema.

Buscar Paquetes

pacman -Ss string1 string2 ...

Busca paquetes en la base de datos de Pacman.

pacman -Qs string1 string2 ...

Busca entre los paquetes ya instalados.

Limpiar Caché de Paquetes

paccache -r

Limpia la caché de paquetes, manteniendo las 3 versiones más recientes.

pacman -Sc

Elimina toda la caché de paquetes que ya no están instalados.

Notas Adicionales

Recuerda que para usar Pacman necesitas permisos de superusuario. Puedes preceder los comandos con 'sudo' o cambiar al usuario root.

Yay es un wrapper de Pacman que también maneja paquetes del AUR (Arch User Repository). La mayoría de los comandos de Pacman funcionan de manera similar en Yay.
